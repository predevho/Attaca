# DOMAIN-COMMON-CONSTITUTION

공통 도메인 원칙. 모든 도메인이 공유하는 기본 원칙을 정의한다.

---

## 1. 공통 개체 표현

* 모든 엔티티는 생성/수정 시각을 갖는다. (`createdAt`, `updatedAt`)
* 삭제는 도메인 성격에 따라 물리 삭제 또는 soft delete를 선택한다. soft delete가 필요한 도메인은 해당 STATUTE에 명시한다.

---

## 2. 요청/응답 원칙

* Entity를 컨트롤러 밖으로 노출하지 않는다. 요청·응답은 DTO로만 주고받는다.
* API 응답은 일관된 형태를 유지한다. 성공과 실패의 형태를 통일한다.

---

## 3. 예외 처리 원칙

* 비즈니스 예외는 공통 예외 체계(`BusinessException` + 에러 코드)로 표현한다.
* 각 계층은 예외를 임의로 삼키지 않는다. 예외 → 응답 변환은 전역에서 일괄 처리한다.

---

## 4. 인증/인가 원칙

* 인증은 JWT를 기본으로 하며, 소셜 로그인(OAuth2)을 함께 지원한다.
* 권한은 최소 두 가지를 둔다: `ROLE_USER`, `ROLE_ADMIN`.
* 관리 기능은 `ROLE_ADMIN` 권한으로만 수행한다.

---

## 5. 목록 조회 원칙

* 목록 조회는 페이징을 기본으로 한다. 무한정 전체 조회를 제공하지 않는다.
